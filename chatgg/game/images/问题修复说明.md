# 角色立绘显示问题修复说明

## 问题历史

### 第一次问题
**描述**: 角色立绘（1080×1440）只显示下半部分，人物被截断。
**原因**: 锚点设置不正确。
**解决**: 修正了锚点设置，使用 (0.5, 1.0) 确保人物底部对齐屏幕底部。

### 第二次问题（当前）
**描述**: 人物太大，超出屏幕范围，没有将整个人物放入。
**原因**: 1440像素的立绘高度 > 1080像素的屏幕高度。
**解决**: 添加缩放功能，使用 `zoom=0.65` 自动缩放。

---

## 游戏配置信息

| 配置项 | 数值 |
|--------|------|
| **游戏分辨率** | 1920×1080 |
| **角色立绘尺寸** | 1080×1440 |
| **文本框高度** | 278像素 |
| **默认缩放比例** | 0.65 |
| **缩放后高度** | ~936像素 (1440×0.65) |

---

## 修复方案

### 1. 新增缩放配置
在 `characters.rpy` 中添加：
```renpy
define char_zoom = 0.65
```

### 2. 位置变换带缩放
```renpy
transform left:
    xpos 0.15
    ypos 1.0
    xanchor 0.5
    yanchor 1.0
    zoom char_zoom
```

### 3. 灵活的缩放参数
`show_char` 函数现在支持可选的 `zoom` 参数：
```python
def show_char(... zoom=None):
    # 使用自定义缩放或默认缩放
```

---

## 使用方法

### 基础用法（使用默认缩放）
```renpy
# 显示沈思在左侧（缩放0.65）
$ show_char("shen_si", "normal", position="left")

# 显示林子轩在中间
$ show_char("lin_zixuan", "nervous", position="center")

# 隐藏角色
$ hide_char("shen_si")
```

### 自定义缩放
```renpy
# 使用更大的缩放（0.75）
$ show_char("shen_si", "normal", position="left", zoom=0.75)

# 使用更小的缩放（0.55）
$ show_char("lin_zixuan", "nervous", position="center", zoom=0.55)
```

### 调整全局缩放
修改 `characters.rpy` 中的 `char_zoom` 值：
```renpy
define char_zoom = 0.70  # 改为更大
# 或
define char_zoom = 0.60  # 改为更小
```

## 图片规格建议

### 角色立绘
- **原始尺寸**: 1080×1440
- **格式**: PNG（支持透明背景）
- **人物位置**: 人物应在图片的下半部分，底部贴合图片底部
- **缩放**: 游戏会自动缩放，无需预先调整图片大小

### 背景图片
- **尺寸**: 1920×1080
- **格式**: PNG 或 JPG

## 调试技巧

如果显示效果不理想：

### 方案1：调整全局缩放
```renpy
# 在 characters.rpy 中修改
define char_zoom = 0.70   # 放大一点
define char_zoom = 0.60   # 缩小一点
```

### 方案2：在剧本中使用自定义缩放
```renpy
# 尝试不同的缩放值
$ show_char("shen_si", "normal", position="left", zoom=0.55)  # 更小
$ show_char("shen_si", "normal", position="left", zoom=0.75)  # 更大
```

### 推荐的缩放范围
| 场景 | 缩放值 | 效果 |
|------|--------|------|
| 近距离特写 | 0.75-0.85 | 人物大，细节清楚 |
| 标准对话 | 0.60-0.70 | 人物适中，平衡 |
| 多人场景 | 0.50-0.60 | 人物较小，节省空间 |
