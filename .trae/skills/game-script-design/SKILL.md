# Game Script Design

游戏剧本设计技能，专为视觉小说与文字冒险游戏打造。

## 核心能力

- **剧本创作**：根据用户提供的关键词、主旨或段落，自由扩充和优化为完整的游戏剧本
- **结构设计**：规划游戏剧本的章节结构、起承转合、高潮与结局
- **角色对话**：塑造角色性格，编写符合角色设定的对话和独白
- **分支剧情**：设计多分支、多结局的剧情选项系统
- **世界观构建**：创建游戏背景设定、场景描述和氛围营造
- **文本优化**：润色现有剧本，提升文字表现力和情感张力

## 工作流程

### 1. 理解需求

首先明确以下要素：

- **游戏类型**：视觉小说 / 文字冒险 / 交互式叙事
- **核心主题**：用户想表达的主旨或情感
- **关键词**：需要融入的核心元素（人物、地点、道具、事件等）
- **风格偏好**：日系 / 韩系 / 欧美 / 古风 / 科幻等
- **篇幅要求**：短篇 / 中篇 / 长篇

### 2. 剧本结构

遵循经典的三幕式结构：

```
第一幕（起）：建立世界，引入角色，抛出核心冲突
    ↓
第二幕（承/转）：冲突升级，角色成长/改变，层层递进
    ↓
第三幕（合）：高潮对决/抉择，结局收束
```

### 3. 角色塑造

每个角色需要明确：

- **基础属性**：姓名、年龄、外貌特征
- **性格特质**：MBTI / 核心性格词 / 说话风格
- **角色弧光**：从开始到结束的变化轨迹
- **关系网络**：与其他角色的亲疏远近

### 4. 对话设计原则

- **性格一致性**：对话符合角色性格和背景
- **信息传递**：推进剧情或揭示角色内心
- **节奏把控**：长短句交错，张弛有度
- **潜台词**：通过省略、暗示增加深度

### 5. 分支设计

```
主线路 ──┬── 结局A（理想结局）
        ├── 结局B（悲剧结局）
        └── 结局C（隐藏结局）
        
支线路 ──┬── 角色专属剧情
        ├── 世界观补充
        └── 小游戏/收集要素
```

## 输出格式

### 剧本大纲

```markdown
# 《游戏名称》剧本大纲

## 基本信息
- 类型：视觉小说/文字冒险
- 主旨：XXX
- 章节数：X章

## 世界观设定
（背景、场景、重要概念）

## 角色一览
（主要角色介绍）

## 章节大纲
### 第一章：标题
- 主要事件
- 关键抉择点
- 角色出场

### 第二章：标题
...
```

### 详细剧本片段

```markdown
# 第一章：清晨

## 场景1：教室

<场景描述>
清晨的阳光透过窗帘洒入教室，空气中弥漫着淡淡的花香。

[主人公]：（内心独白）又是平凡的一天...

> 选择分支
> A. 看向窗外发呆
> B. 和同桌说话
> C. 翻开课本预习
```

### 角色设定卡

```markdown
## 角色名：XXX

- **性别**：
- **年龄**：
- **性格**：开朗/内向/傲娇...
- **声线**：温柔/低沉/活泼...
- **角色弧光**：从XXX到XXX的转变
- **代表色**：
- **口头禅**：
```

## 使用示例

### 示例1：基础剧本生成

用户输入：
> 关键词：校园、悬疑、隐藏的秘密
> 主旨：真相与谎言的边界
> 风格：日系轻小说

输出：完整的剧本大纲、角色设定、第一章详细剧本

### 示例2：段落扩充

用户输入：
> "她站在窗边，望着远方的天空。"

输出：扩充为包含场景描写、角色动作、内心独白、氛围铺垫的完整段落

### 示例3：对话优化

用户输入：
> 角色A：你好
> 角色B：你好

输出：根据角色性格和情境重写为更具表现力和角色特色的对话

### 示例4：分支设计

用户输入：
> 场景：主角发现了一封神秘的信

输出：设计3-5个不同方向的分支选项，每个选项对应不同的剧情走向

## 注意事项

- 始终以用户提供的关键词和主旨为核心进行创作
- 保持剧情逻辑自洽，避免矛盾
- 角色对话要符合其性格设定和成长轨迹
- 分支选项要有明确的意义差异，影响玩家体验
- 注意情感节奏，张弛有度，避免全程高能或全程平淡
